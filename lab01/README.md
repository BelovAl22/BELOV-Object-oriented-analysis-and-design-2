Приложение представляет собой графическую систему управления заметками, реализованную на Windows Forms.

В состав системы входят следующие классы:

Program — точка входа в приложение
Form1 — главное окно со списком заметок
EditForm — окно редактирования заметки
NoteManager — класс управления заметками

2. Реализация класса NoteManager (Singleton)

Класс NoteManager отвечает за хранение и управление списком заметок.

В версии с использованием паттерна Singleton гарантируется существование только одного экземпляра данного класса в пределах приложения.

Основные особенности реализации:
Конструктор объявлен как private, что запрещает создание объектов через оператор new вне класса.
Внутри класса объявлено статическое поле instance, в котором хранится единственный экземпляр.
Доступ к объекту осуществляется через статический метод GetInstance().
Механизм работы следующий:
При первом вызове GetInstance() создаётся новый объект.
При последующих вызовах возвращается уже существующий экземпляр.

3. Использование Singleton в главной форме (Form1)

Главная форма предназначена для:

отображения списка заметок
добавления новых заметок
удаления заметок
открытия окна редактирования
Вместо создания нового экземпляра NoteManager используется вызов: NoteManager.GetInstance()

Таким образом, форма получает ссылку на единственный существующий объект менеджера заметок.
Это исключает возможность создания нескольких независимых списков заметок.

4. Использование Singleton во второй форме (EditForm)

Вторая форма предназначена для редактирования выбранной заметки.

Аналогично главной форме, объект NoteManager не создаётся заново, а получается через вызов GetInstance().

Это гарантирует, что:
обе формы работают с одним и тем же списком данных
изменения, выполненные во второй форме, немедленно отражаются в главной
Таким образом, обеспечивается единый источник данных (Single Source of Truth) в рамках приложения.

5. Решаемая проблема

Без использования паттерна Singleton существует риск:
создания нескольких экземпляров NoteManager
рассинхронизации данных между формами
нарушения целостности состояния приложения

Применение паттерна Singleton устраняет данную проблему, обеспечивая:
централизованное хранение данных
контроль жизненного цикла объекта
гарантированное наличие только одного экземпляра менеджера

6. Вывод

В рамках разработанного приложения паттерн Singleton используется для управления глобальным состоянием системы — списком заметок.
Хотя в небольших проектах возможно обойтись передачей зависимостей через конструктор, применение Singleton демонстрирует механизм централизованного контроля доступа к общему ресурсу и решает проблему потенциальной рассинхронизации данных при расширении системы.
